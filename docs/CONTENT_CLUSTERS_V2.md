## Content clusters v2 (controlled, non-cannibalizing)

Purpose: keep **one primary intent per URL** (pillar pages), and use blog posts only for **supporting long-tail** questions that naturally link back to the pillar + booking pages.

This prevents the “many similar blogs competing with each other” problem and helps Google understand the authority structure.

### Pillar pages (primary targets)

- **Pricing / tariff**: `/room-prices` and `/hi/room-prices`
- **Amenities / meals included**: `/amenities` and `/hi/amenities`
- **Group / yatri mandal**: `/group-booking` and `/hi/group-booking`
- **Photos / trust**: `/gallery` and `/hi/gallery`
- **Reviews (verified source only)**: `/reviews` and `/hi/reviews`

### Redirect policy (anti-cannibalization)

If a blog post’s *main purpose* is “room prices”, “amenities list”, “group booking”, or “reviews”, it should **not** exist as a blog primary page.
Instead, redirect the blog URL → the pillar URL.

Redirects already added (examples):

- `/blog/janaki-mahal-trust-room-prices-2025-all-types` → `/room-prices`
- `/blog/sri-janaki-mahal-trust-room-prices-pricing-guide` → `/room-prices`
- `/blog/shri-janki-mahal-amenities-facilities-list` → `/amenities`
- `/blog/sri-janaki-mahal-trust-services-amenities-facilities` → `/amenities`
- `/blog/sri-janaki-mahal-trust-group-booking` → `/group-booking`
- `/blog/janaki-mahal-trust-reviews-guest-testimonials` → `/reviews`
- `/blog/sri-janaki-mahal-trust-reviews-and-ratings` → `/reviews`
